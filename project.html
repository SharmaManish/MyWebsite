COMPLETED PROJECTS



1. Clustering of Data Streams

Discipline: Clustering of Data streams and Anomaly Detection

The project aims at the development of a new algorithm for

1) Clustering of Data streams in efficient way that having less loss of quality at cost of time complexity.
2) Detecting Anomalies in data streams and use of variable size buckets for storing data.
3) Deletion of data from the clustered data to empty space for recently arrived data.

Currently one research paper has been written with the above defined characteristics to simplify the clustering problems .
The paper got published in 1st International Conference RAIT 2012 held in ism dhanbad from 15-17 march 2012.

Paper - Title- pre-clustering approach
This paper deals with clustering of data as well as anomaly detection with variable size buckets. The proposed algorithm allows data and assign to particular cluster based on Euclidean distance. The filled clusters enter into buckets and buckets are allotted to the clusters based on Euclidean distance.

Mentor: Dr Durga Toshniwal, Electronics and Computer Engineering Department, IIT Roorkee


2. Configure a unix machine to behave like a firewall

Discipline: Unix Firewall and Malicious attacks

The project aims at configuring of firewall by creating two scripts

1) Writing a script with iptables and make unix system secure and free from malicious attacks.
2) Scapy tool is used to generate attacks and a attacker script is created for generating attacks.

Attacker Script: TCP SYN Queue Flood is generated against the target machine, it is a type of DOS (Denial of Service) attack, it took advantage of TCP “Three Way Handshake” protocol. Client sends TCP SYN Packet to target machine, it replies with SYN ACK packet, but client doesn’t respond leaving the connection open. Multiple Attackers leave the connection half open and creates load and make it unavailable for legitimate users.
Prevention Script: Shell script written to reject all suspicious TCP connections, it allows only 25 attempts from a ip address after that forcefully reject the connection.

Mentor: - Dr. Anjali Sardana, Electronics and Computer Engineering Department, IIT Roorkee



INTERNSHIP Microsoft



Approach for Scalability Testing of Azure Services

Discipline: Developing a Tool for Scalability Testing of Cloud Services.

The Project aims at

1) Identify and prioritize the set of requirements in Scalability Testing Tool.
2) Evaluate the currently most promising tools used for scalability testing by Microsoft and identify which meet nearest to the requirements.
3) Implement a P0 functionality missing in the tool.

Identification and Evaluation of Tools: - The initial phase of project includes evaluation of capability of four currently existing and most promising tools of Microsoft, which are- ATE, Load Test Service, CloudTestHost and Tip. Deployment and Scalability testing by writing test cases for the four tools and report bugs in the tools. Prepare a comparison matrix of the four tools against the gathered requirements and select one of the tools which meet nearest to requirements.

Implementing P0 functionality: - Selected Load Test Service as most promising tool. Developed feature for tracking live test progress of test results.

Mentor: - Rochak Mittal, Test Lead II in Microsoft India Development Center. Company- Microsoft India Development Center.
